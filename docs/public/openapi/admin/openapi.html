X-Request-Id

X-Request-Id 是一个 HTTP 头字段，用于唯一标识每个请求。它在分布式系统和微服务架构中非常有用，可以帮助跟踪请求在多个服务之间的流动情况，便于日志记录、调试和问题排查。 X-Request-Id 应该是全局唯一的，建议使用 ULID 生成的字符串来实现。

作用

  • 请求跟踪：X-Request-Id 使得每个请求在系统的各个组件之间传递时都有一个唯一的标识符，可以方便地跟踪请求的整个生命周期。
  • 日志关联：通过在日志中记录 X-Request-Id，可以将不同服务的日志条目关联起来，帮助快速定位和解决问题。
  • 性能监控：结合 X-Request-Id 和其他监控工具，可以更准确地测量请求的处理时间和服务间的延迟。
  • 调试和故障排除：当出现问题时，可以通过 X-Request-Id 快速找到相关的日志和数据，从而更快地进行故障排除。

使用方法

  • 客户端发送请求：客户端在发送请求时，可以在 HTTP 头中添加 X-Request-Id 字段。如果客户端没有提供 X-Request-Id，服务端可以自动生成一个并返回给客户端。
  • 服务端接收请求：服务端接收到请求后，会读取 X-Request-Id 并在整个请求处理过程中保持该标识符不变。服务端还可以在响应头中返回 X-Request-Id，以便客户端在后续请求中使用相同的标识符。
  • 日志记录：在每个服务的日志中记录 X-Request-Id，确保每个日志条目都包含这个标识符，以便于后续的分析和关联。

示例

客户端发送请求

GET /api/resource HTTP/1.1
Host: example.com
X-Request-Id: 01H9JQ23GZ54V8KXWYB6CZT3D

服务端响应

HTTP/1.1 200 OK
Content-Type: application/json
X-Request-Id: 01H9JQ23GZ54V8KXWYB6CZT3D

{
  "data": "some data"
}

x-permission-code

权限代码 (x-permission-code) 是用于标识特定权限的字符串。这些权限代码用于系统的权限管理，以确保用户只能访问他们被授权的资源。

权限代码格式

权限代码通常由两部分组成：

  1. 权限名称：表示权限的唯一标识符。
  2. 操作类型：表示对资源的操作类型，如读取(R)、写入(W)和删除(D)。

例如，权限代码 DICT:R 表示对字典管理的读取权限。

X-API-Version

X-API-Version 是用于声明接口主版本号的 HTTP 请求头。通过该请求头，客户端可以在不变更 URL 的情况下选择期望的 API 行为版本。

取值规则

  • 可选请求头：客户端可在请求头中传入 X-API-Version。
  • 默认版本：当未传该请求头时，服务端按默认版本 1 处理。
  • 支持版本：当前仅支持 1。
  • 非法值处理：非整数或不支持的版本号会返回参数错误。

使用建议

  • 客户端应固定传递明确的版本号，避免因默认版本变更带来行为差异。
  • 仅在存在兼容性差异时升级版本号；同一主版本内保持兼容。

示例

GET /me/profile HTTP/1.1
Host: example.com
X-API-Version: 1
Authorization: Bearer eyJ...

我的通知增量轮询

高频轮询增量接口，返回游标之后的变化集合。 可见性规则与 /me/notifications 一致：

  • status = 200
  • (expire_at IS NULL OR expire_at > now())
  • scope=2 全局，或 scope=1 命中当前管理员

游标规则：

  • 首次请求不传 cursor
  • 后续请求使用上次响应中的 next_cursor
  • 服务端按 (updated_at DESC, id DESC) 稳定排序
