Mahu 文档站点默认集成了 `nuxt-llms` 模块，会自动生成 `/llms.txt` 与 `/llms-full.txt` 两个文件，供大模型直接索引文档内容。

## 适用对象

- 需要把 Mahu 文档暴露给大模型抓取、建立外部索引或喂给知识库的维护者。
- 需要排查 `llms.txt`、`llms-full.txt` 链接是否正确的文档站点维护者。

## 当前项目如何生成

当前仓库没有依赖模块默认推断域名，而是在 `docs/nuxt.config.ts` 中显式配置了 `llms`：

- `domain`：优先读取 `NUXT_SITE_URL`，其次读取 `NUXT_PUBLIC_SITE_URL`
- 如果两个环境变量都没提供，则回退到占位域名 `https://mahu-docs.invalid`
- `title` 与 `description` 使用仓库当前的 Mahu 文档站点文案，而不是模板站点文案

## 自定义配置

可在 `nuxt.config.ts` 中覆盖 LLM 索引信息：

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  llms: {
    domain: 'https://your-docs-url',
    title: 'Mahu 开发文档',
    description: '面向 Mahu 内部开发者的项目总览、架构设计与数据库规范入口。',
    full: {
      title: 'Mahu 开发文档（完整版）',
      description: '面向 Mahu 内部开发者的项目总览、架构设计与数据库规范入口。',
    },
  },
})
```

## 构建产物

执行 `cd docs && pnpm build` 后，可重点检查以下文件：

- `docs/.output/public/llms.txt`
- `docs/.output/public/llms-full.txt`

它们中的链接域名必须指向 Mahu 文档站，而不是模板站点或本机文件路径。

## 常见误区

- 没有设置站点 URL 就直接发布，会让生成出来的 `llms.txt` 指向占位域名。
- 把模板示例域名保留在配置里，会让 AI 抓取结果错误地跳到外部站点。
- 在文档正文中写本机绝对路径链接，会把错误链接继续扩散到 `llms-full.txt`。

## 与 MCP 的区别

- `llms.txt` / `llms-full.txt`：适合静态抓取、批量索引、离线喂给知识库。
- MCP：适合 IDE / Agent 在对话期间按需实时读取文档。
