本章节用于说明 Mahu 文档如何接入各类 AI 工具。这里不讨论业务模型或提示词设计，只聚焦文档站点本身暴露给 IDE、Agent 和大模型的两条入口。

## 两类入口

- **MCP Server**：面向 Claude Code、Cursor、VS Code 等支持 MCP 的工具，适合在对话过程中实时读取文档。
- **LLM 索引文件**：面向批量抓取或离线索引，适合通过 `llms.txt` / `llms-full.txt` 让大模型一次性发现站点内容。

## 推荐阅读顺序

1. 先看「[MCP 接入 Mahu 文档](/ai/mcp)」，确认实时访问入口如何接入工具。
2. 再看「[LLM 集成与索引](/ai/llms)」，确认静态索引文件如何生成与校验。

## 使用边界

- 如果你的目标是“让工具在回答时随时拉文档”，优先用 MCP。
- 如果你的目标是“让搜索、知识库或外部模型先批量收录文档”，优先检查 LLM 索引文件。
- 两类入口都依赖正确的站点域名配置，因此发布前要先确认 `nuxt.config.ts` 中的站点 URL 来源正确。
