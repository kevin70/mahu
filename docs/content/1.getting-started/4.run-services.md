## 适用对象

- 第一次在本地启动 Mahu 的开发者。
- 需要确认管理端 API 与任务进程是否都已正常运行的维护者。

## 启动前检查

启动应用前，建议先确认：

- PostgreSQL 已启动并可访问。
- 如需文件相关功能，MinIO 已启动。
- 已完成一次 `./gradlew build`。
- 如需改动管理端接口，已执行 `./gradlew :mahu-admin:mahu-admin-web:openApiGenerate`。

## 启动管理端应用

管理端入口为 `mahu-admin:mahu-admin-web`，主类为 `cool.houge.mahu.admin.AdminApplication`。

```bash [Terminal]
./gradlew :mahu-admin:mahu-admin-web:run
```

默认端口来自 `mahu-admin/mahu-admin-web/src/main/resources/application.yaml`：

- HTTP 端口：`9090`

最基础的本地数据库配置也在该文件中，默认指向本机 PostgreSQL。

## 启动任务服务

任务进程入口为 `mahu-task`，主类为 `cool.houge.mahu.task.TaskApplication`。

```bash [Terminal]
./gradlew :mahu-task:run
```

任务服务同样依赖数据库配置，默认从 `mahu-task/src/main/resources/application.yaml` 读取本地 PostgreSQL 连接信息。

## 如何验证是否启动成功

可以从以下几个角度确认：

1. 日志中出现应用启动完成信息。
2. 管理端监听 `9090` 端口。
3. 任务服务日志中能够看到调度器注册或 worker 启动相关日志。
4. 如数据库不可用，应用通常会在启动阶段直接报错退出。

## 常见问题

### PostgreSQL 未启动或连接信息不匹配

常见表现：

- 启动阶段报 JDBC 连接失败。
- EBean 或数据源初始化失败。

优先检查：

- PostgreSQL 容器是否在运行。
- 端口、数据库名、用户名、密码是否与 `application.yaml` 对齐。

### MinIO 未启动

如果你当前在调试文件上传、对象存储转发等功能，需要额外确认 MinIO 已就绪；否则相关接口会在运行期失败。

### OpenAPI 生成代码未更新

如果你刚改过 `openapi/admin/**` 下的规范文件，但管理端编译失败，优先执行：

```bash [Terminal]
./gradlew :mahu-admin:mahu-admin-web:openApiGenerate
```

### 端口冲突

如果 `9090` 已被其他进程占用，管理端应用会启动失败。优先释放冲突端口，或为本地调试提供单独配置覆盖。
