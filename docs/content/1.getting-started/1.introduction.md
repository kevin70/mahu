Mahu 是一个以 Java/Helidon SE 为核心的后端工程，仓库内包含多个业务与基础设施模块；同时提供本文档站点，用于沉淀架构规范、数据库约定与 AI 集成方式。

## 当前现状

- 这是一个 Gradle 多模块仓库，核心运行应用为管理端 API 服务与独立任务服务。
- 接口契约通过 OpenAPI 管理，管理端代码由 OpenAPI 生成骨架后再由手写 Controller 实现。
- 数据库以 PostgreSQL 为主，结构与初始化数据统一由 `mahu-db` 中的 Liquibase 入口文件管理。

## 技术栈

| 分类 | 技术 |
|------|------|
| 语言 / 运行时 | Java 25 |
| Web 框架 | Helidon SE 4.4 |
| ORM | EBean 17 |
| 数据库 | PostgreSQL 17 |
| 数据库迁移 | Liquibase |
| 对象存储 | MinIO |
| 定时任务 | db-scheduler |
| 本地缓存 | Caffeine |
| 依赖注入 | Helidon Service Registry（编译时） |
| 构建工具 | Gradle（多模块） |
| API 规范 | OpenAPI 3.0 |
| OpenAPI 工具链 | Redocly + OpenAPI Generator |
| 代码格式 | Palantir Java Format |

补充说明：

- `Valkey` 依赖已经在部分模块中预留，但不应在项目概览中直接表述为当前统一缓存方案。
- 目前从仓库现状无法确认 `RabbitMQ` 已作为运行时核心组件接入，因此不在总览技术栈中列为既成事实。

## 可运行应用

仓库当前有两个主要运行应用：

- `mahu-admin:mahu-admin-web`：管理端 API 应用，提供后台管理相关接口。
- `mahu-task`：独立任务进程，负责延时任务与后台调度处理。

## 模块一览

仓库以 Gradle 多模块形式组织（以 `settings.gradle` 为准）：

| 模块 | 说明 |
|------|------|
| `mahu-bom` | Java Platform / BOM，统一依赖版本（配合 `gradle/libs.versions.toml`） |
| `mahu-base` | 基础设施与通用工具 |
| `mahu-config` | 配置模型、配置前缀与配置相关枚举 |
| `mahu-entity` | EBean 实体与实体相关模型 |
| `mahu-model` | 非持久化业务模型；当前主要承载查询对象 |
| `mahu-codegen` | 代码生成相关工具 |
| `mahu-db` | Liquibase 入口、Schema 迁移与初始化数据脚本 |
| `mahu-dal` | 数据访问层（Repository 实现，基于 EBean） |
| `mahu-shared` | 跨应用共享的领域/应用服务 |
| `mahu-web` | 通用 HTTP/Web 能力（Helidon WebServer 封装、错误处理） |
| `mahu-admin:mahu-admin-infra` | 管理端基础设施与业务服务层 |
| `mahu-admin:mahu-admin-web` | 管理端 Helidon SE API 应用（基于 OpenAPI 生成） |
| `mahu-task` | 独立定时任务服务（db-scheduler） |
| `openapi` | OpenAPI 规范源文件、Redocly 打包与文档产物 |
| `docs` | 开发文档站点（Nuxt + Docus） |

## Java Package 规范

包结构以功能后缀命名，各层职责如下：

| 包名 | 说明 |
|------|------|
| `util` | 工具类 |
| `entity` | 数据实体，与数据库表一一对应 |
| `repository` | 数据库访问对象（类名以 `Repository` 结尾） |
| `service` | 业务服务（类名以 `Service` 结尾） |
| `shared` | 跨模块共享的内部公共业务逻辑 |
| `controller` | REST 接口（类名以 `Controller` 结尾） |
