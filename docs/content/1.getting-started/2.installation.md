## 前置条件

- **JDK 25**（推荐 [Eclipse Temurin](https://adoptium.net/zh-CN)）
- **Git**
- **Node.js / pnpm**（用于 `docs/` 文档站点）
- **Docker**（用于启动本地依赖服务）

具体版本以仓库根目录 `README.md` 与根 `build.gradle` 为准。

## 拉取代码

```bash [Terminal]
git clone <your-repo-url>
cd mahu
```

## 启动本地依赖服务

使用 Docker 启动 PostgreSQL 和 MinIO。以下示例与
`mahu-admin/mahu-admin-web/src/main/resources/application.yaml` 和
`mahu-task/src/main/resources/application.yaml` 的默认本地配置对齐：

- PostgreSQL 地址：`127.0.0.1:5432`
- 数据库名：`mahu`
- 用户名：`postgres`
- 密码：`postgres`

### PostgreSQL

```bash [Terminal]
docker run -d --name postgres \
  -p 5432:5432 \
  -e POSTGRES_DB=mahu \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  postgres:17.5-alpine \
  postgres -c fsync=off
```

如果你本地已经占用了 `5432`，或者希望使用其他数据库名/账号，必须同步覆盖两个运行模块里的
`db.url`、`db.username`、`db.password`，否则后续 `run` 命令会直接连错库。

### MinIO

```bash [Terminal]
docker run -d --name minio \
  -p 19000:9000 \
  -p 19001:9001 \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin \
  minio/minio:RELEASE.2025-05-24T17-08-30Z \
  server --console-address :9001 /data
```

MinIO 控制台地址：`http://127.0.0.1:19001`，默认账号 `minioadmin / minioadmin`。

## 构建与测试（后端）

在项目根目录执行：

```bash [Terminal]
./gradlew build
```

Windows PowerShell 使用 `.\gradlew.bat`：

```powershell [Terminal]
.\gradlew.bat build
```

单模块构建/测试示例：

```bash [Terminal]
./gradlew :mahu-web:build
./gradlew :mahu-entity:test
```

## 安装 OpenAPI 前端依赖

首次开发或更新 OpenAPI 规范前，需先安装前端依赖：

```bash [Terminal]
./gradlew openapiDepsInstall
```

## 生成管理端 API 代码

```bash [Terminal]
./gradlew :mahu-admin:mahu-admin-web:openApiGenerate
```

生成结果写入 `mahu-admin/mahu-admin-web/src/main/gen`，勿手动修改。

## 启动文档站点（docs）

```bash [Terminal]
cd docs
pnpm install
pnpm dev
```

文档内容在 `docs/content/`，修改后热更新。构建：

```bash [Terminal]
cd docs
pnpm build
```

## JVM 启动参数（生产参考）

```
-Xms2g -Xmx2g
-XX:MetaspaceSize=128m -XX:MaxMetaspaceSize=256m
-XX:+UseZGC
-Xlog:gc*=info:file=/logs/gc.log:time,uptime,pid:filecount=5,filesize=100M
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/logs/heapdump.hprof
-XX:+DisableExplicitGC
```
