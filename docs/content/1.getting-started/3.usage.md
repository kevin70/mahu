本文档站点使用 Nuxt Content（MDC）渲染 `docs/content/` 下的 Markdown。页面、导航与组件用法请遵循本仓库约定，避免把模板示例与外部链接带入内部规范文档。

## 适用对象

- 需要新增、调整或重写 `docs/content/` 页面内容的开发者。
- 需要排查导航缺失、分区入口不显示或站内链接失效问题的维护者。

## 内容目录与命名

- **目录**：所有内容页面放在 `docs/content/`。
- **章节目录**：使用编号前缀保证排序，例如 `1.getting-started/`、`2.architecture/`。
- **章节内页面**：使用编号前缀 + kebab-case，例如 `1.introduction.md`、`2.installation.md`。

## 页面 frontmatter（必需）

每个页面都应包含 `title` 与 `description`，首页和各章节入口页也不例外：

```yaml [page.md]
---
title: 页面标题
description: 该页的简短说明
---
```

如需在侧边栏展示图标，可加 `navigation.icon`（Lucide）：`i-lucide-...`。
如果页面需要作为路由存在、但不显示在侧边栏中，使用 `navigation: false`，这通常用于首页或分区 `index.md`。

## 导航（.navigation.yml）

每个章节目录应包含 `.navigation.yml`，至少包含：

```yaml [.navigation.yml]
title: 章节标题
icon: i-lucide-book-open
```

## 站内链接约定

- 站内文档跳转统一使用从内容根开始的绝对路径，例如 `/getting-started/installation`。
- 不要写 `../page.md` 这类文件系统相对路径，也不要把本机源码路径写成链接目标。
- 引用仓库内源码、SQL 或配置文件时，优先写成行文中的路径文本，例如
  `mahu-base/src/main/java/.../Metadata.java`，而不是 `/Users/...` 绝对路径。

## MDC 组件注意事项

- Nuxt UI 组件在 MDC 中必须使用 `u-` 前缀（例如 `::u-page-hero`、`:::u-button`）。
- Nuxt Content 的通用组件（例如 `::note`、`::tip`、`::code-group`）不需要 `u-`。

## 常见误区

- 首页或分区入口页只写 `seo` 不写 `title` / `description`，会让页面规范与实际站点规则脱节。
- 直接复制 IDE 中的绝对文件路径，会让最终 HTML 和 `llms-full.txt` 生成不可访问的伪链接。
- 在 `::code-group` 里混放命令和配置文件，不利于读者理解步骤顺序。
