## 一句话理解

Mahu 是一个以 `mahu-admin-web` 管理端 API 服务和 `mahu-task` 任务服务为核心的 Gradle 多模块后端仓库，底层通过 `mahu-domain`、`mahu-dal`、`mahu-shared` 等模块复用领域模型、数据库访问和共享业务能力。

## 模块关系图

```text
openapi ----> mahu-admin-web <---- mahu-web
                  ^                 ^
                  |                 |
            mahu-admin-infra ---- mahu-shared
                  ^                 ^
                  |                 |
                mahu-dal -----------+
                  ^
                  |
              mahu-domain
                  ^
                  |
               mahu-base

mahu-db 为数据库迁移与初始化数据入口
docs 为开发文档站点
mahu-task 复用 mahu-shared / mahu-dal 执行后台任务
```

## 按层理解仓库

### 基础层

- `mahu-base`：基础工具、异常、分页、仓储基类。
- `mahu-domain`：实体、查询对象、配置模型与领域枚举。

### 数据与共享能力层

- `mahu-dal`：Repository 与数据库访问实现。
- `mahu-shared`：跨应用共享的缓存、对象存储、通用业务服务。
- `mahu-web`：通用 Web 平台能力。

### 应用层

- `mahu-admin:mahu-admin-infra`：管理端专用业务服务与基础设施。
- `mahu-admin:mahu-admin-web`：管理端 API 应用。
- `mahu-task`：任务与调度应用。

### 支撑资产

- `mahu-db`：Liquibase 入口、结构迁移与初始化数据。
- `openapi`：接口契约、Redocly 打包与文档输出。
- `docs`：面向开发者的文档站点。

## 两个运行应用

### `mahu-admin:mahu-admin-web`

职责：

- 对外提供管理端 API。
- 实现 OpenAPI 生成的服务接口。
- 组合 `mahu-admin-infra`、`mahu-shared`、`mahu-web` 暴露业务能力。

### `mahu-task`

职责：

- 独立执行延时任务与后台 worker。
- 轮询数据库中的待处理任务。
- 通过 topic handler 将异步处理与同步请求解耦。

## 三条最重要的开发链路

1. 接口链路
   `openapi` -> `mahu-admin-web` Controller -> `mahu-admin-infra` Service -> `mahu-dal` Repository -> PostgreSQL

2. 任务链路
   `mahu-task` Worker -> handler -> `mahu-shared` / `mahu-dal` -> PostgreSQL

3. 结构演进链路
   `mahu-db` 迁移脚本 -> `mahu-domain` 实体 -> `mahu-dal` 查询条件 -> 上层业务与测试
