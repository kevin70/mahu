## 当前现状

仓库中的运行应用至少包含以下配置入口：

- `mahu-admin/mahu-admin-web/src/main/resources/application.yaml`
- `mahu-task/src/main/resources/application.yaml`
- 若模块测试有专用配置，还会在 `src/test/resources/` 下放测试配置文件。

另外，部分模块会使用 `meta-config.yaml` 提供配置元信息或测试期配置描述。

## 配置文件角色

### `application.yaml`

用途：

- 定义应用默认运行配置。
- 包含数据库、端口、功能开关调度配置等。

示例：

- `mahu-admin-web` 中声明了 `server.port=9090`
- `mahu-admin-web` 与 `mahu-task` 都声明了本地数据库连接信息

### `meta-config.yaml`

用途：

- 为配置元数据、测试环境或模块化配置约束提供支撑。
- 在阅读某个模块配置行为时，可以把它视为 `application.yaml` 之外的补充说明。

## 常见配置前缀

- `info.*`：应用名、环境等基础元数据。
- `server.*`：端口、关闭钩子、观察能力等服务端配置。
- `db.*`：数据库连接池与连接信息。
- `scheduling.*`：调度、定时刷新、worker 注册相关配置。
- `cors.*`：跨域配置。

### `db.*` 常用连接池参数

- `db.url`：JDBC 连接串。
- `db.username` / `db.password`：数据库连接凭据。
- `db.min-idle`：最小空闲连接数，用于控制启动预热和低峰期保留连接数量。
- `db.max-size`：连接池最大连接数，决定应用可向数据库施加的最大并发。

其余连接池调优参数（如等待超时、空闲回收时间、连接最大生命周期、statement cache 等）当前固定在各应用代码中，不通过共享配置暴露，避免环境之间随意漂移。

## 本地开发最小配置集

如果只是本地启动与联调，通常至少需要确认：

1. `db.url`
2. `db.username`
3. `db.password`
4. `server.port`（仅 Web 应用）

涉及对象存储、鉴权、加密或调度时，再根据对应模块配置补充。

## 环境覆盖建议

文档当前不假设仓库已经统一落地了完整的多环境配置目录结构，但建议遵循下面原则：

- 默认值放 `application.yaml`。
- 本地临时调试使用本地覆盖方式，不直接污染默认配置。
- 敏感值不要硬编码在共享文档示例中。
- 配置 key 统一使用 kebab-case，并按领域分组。

## 改配置时建议同时检查

1. 是否影响 `mahu-admin-web` 和 `mahu-task` 两个应用。
2. 是否需要同步更新 `meta-config.yaml` 或测试资源。
3. 是否涉及文档中提到的默认端口、默认依赖服务。
4. 是否需要为新配置补充 OpenAPI、可观测性或运行文档说明。
