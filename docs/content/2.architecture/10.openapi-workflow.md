## 工作流概览

Mahu 的管理端接口采用“契约优先”方式维护：

1. 在 `openapi/admin/**` 中维护 OpenAPI 源文件。
2. 使用 Redocly 进行校验与 bundle。
3. 由 root 项目执行 Redocly 校验与 bundle，再由 `mahu-admin:mahu-admin-web` 调用 OpenAPI Generator 生成服务端骨架。
4. 在手写 Controller 中实现生成出来的接口。

## 关键目录

- `openapi/admin/openapi.yaml`：管理端 OpenAPI 根入口。
- `openapi/admin/paths/*.yaml`：按接口拆分的 path 文件。
- `openapi/admin/components/**/*.yaml`：参数、schema 等公共组件。
- `openapi/dist/admin/openapi.yaml`：bundle 后的产物。
- `mahu-admin/mahu-admin-web/src/main/gen`：生成代码目录。

## 常用命令

### 安装 OpenAPI 前端依赖

```bash [Terminal]
./gradlew openapiDepsInstall
```

### 打包管理端规范

```bash [Terminal]
./gradlew redoclyBundleOpenApi
```

### 生成服务端骨架代码

```bash [Terminal]
./gradlew :mahu-admin:mahu-admin-web:openApiGenerate
```

## 生成后会发生什么

`openApiGenerate` 会：

- 读取 `openapi/dist/admin/openapi.yaml`
- 使用 `java-helidon-server` 生成器
- 将筛选后的接口与模型复制到 `src/main/gen`

项目中真正的业务实现并不写在生成目录里，而是放在手写的 Controller / Service 中。

## 手写代码与生成代码如何配合

推荐理解方式：

- 生成代码：定义接口签名、模型、HTTP 契约骨架。
- 手写 Controller：实现 `H*Service` 接口，负责参数解析与响应组装。
- Service / Repository：承接业务逻辑与数据访问。

这意味着：

- 可以放心重新生成 `src/main/gen`
- 不应手动修改生成目录中的文件
- 业务逻辑应始终放在手写层

## 何时需要重新生成

出现以下情况时，通常都要重新执行生成：

1. 修改了 `openapi/admin/openapi.yaml`
2. 修改了 `paths/*.yaml`
3. 修改了 `components/schemas/*.yaml`
4. 升级了 OpenAPI Generator 模板或版本

## 常见问题

### 改了规范但 Java 编译没同步

优先执行：

```bash [Terminal]
./gradlew :mahu-admin:mahu-admin-web:openApiGenerate
```

### 生成代码被手工改动

生成目录应视为可再生文件。如果某段逻辑只能靠手改 `src/main/gen` 才成立，说明这段逻辑应该迁回手写 Controller、Service 或模板层。
