## 一致性目标

- `mahu-db` 中的 Liquibase SQL 是 **数据库结构的唯一事实来源**。
- `mahu-entity` 中的 EBean 实体需与对应表结构在以下方面保持一致：
  - 表名与 schema（`@Table(schema = ..., name = ...)`）。
  - 主键、唯一约束、索引涉及的字段。
  - 关键字段的含义与注释。

## 变更流程约定

对数据库表做更改时，推荐遵循下面的顺序：

1. **先改 SQL**：在 `mahu-db/src/main/resources/migrations/` 或 `mahu-db/src/main/resources/data/` 下新增或修改对应文件，并提供清晰的变更与回滚策略。
2. **再改实体**：在 `mahu-entity` 中同步更新对应实体类的字段/注解/文档注释。
3. **再改查询模型**：在 `mahu-model` 中同步更新对应查询对象。
4. **最后改业务**：在 `mahu-admin-infra` 等业务模块里调整使用逻辑与测试。

任何涉及字段新增/删除/重命名/类型变更的需求，都应在 Code Review 中检查这四步是否完整。

## Liquibase 入口与 includeAll

Mahu 的 Liquibase 入口文件为：`mahu-db/src/main/resources/changelog-root.yaml`。

该文件通过 `includeAll` 引入：

- `migrations/`：Schema 结构变更脚本（包含 `changeset`，必须提供可行 `rollback`）。
- `data/`：数据初始化/字典等脚本（同样建议为变更提供可回滚策略，便于环境重建与故障恢复）。

边界建议：

- `migrations/` 放表、列、索引、约束等结构变更。
- `data/` 放初始化数据、基础字典、内置账号/角色等数据脚本。
- 如果一个改动同时包含结构和初始化数据，建议拆成两个文件，分别落在对应目录中。

## Repository 与 Q* 查询一致性（mahu-dal）

`mahu-dal` 的 Repository（以及其内部使用的 Q* 条件查询）会强绑定到实体与表结构的字段、类型与语义。

当 Liquibase 变更导致表字段/约束/含义发生变化时，需同步检查并更新：

- `mahu-dal` 中对实体字段的映射与引用（例如 Q* 条件 predicate 中用到的字段）。
- 基于字段语义构建的条件更新/筛选逻辑（例如状态字段、时间字段的比较条件）。

在 Code Review 阶段，把“DB 变更 -> entity 字段 -> dal 查询条件”作为一条检查链路，避免出现文档/实体更新了但查询条件仍沿用旧字段或旧约束的情况。

## 一致性校验（建议）

为减少人为疏漏，后续可以在 CI 或开发脚本中引入轻量的一致性校验，例如：

- 启动一个临时数据库，应用 `mahu-db` 最新的 Liquibase changelog。
- 通过 EBean 或简单脚本读取当前数据库的表列表，与 `@Table(schema, name)` 标注的实体进行比对，找出：
  - 实体引用的表在数据库中不存在；
  - 数据库中已存在表，但尚未有对应实体（如需映射，应补充；如不需要，则可在文档中标明原因）。

本文件只约定原则与流程，具体脚本与 CI 集成可以在需要时逐步补充。这样既不强行引入复杂工具，又为后续自动化留出了清晰的落脚点。

## 变更联动检查清单

当你修改数据库表时，至少检查下面几个位置：

1. `mahu-db`
   新增或更新对应迁移文件，确认目录、命名、回滚策略正确。
2. `mahu-entity`
   更新实体上的 `@Table`、字段与注释。
3. `mahu-model`
   更新相关 query 对象。
4. `mahu-dal`
   检查 Repository 与 QBean 查询条件、排序、条件更新是否仍然正确。
5. 初始化数据
   如果新增了非空字段或新增基础表，检查 `data/` 中的脚本是否需要同步补值。
6. 测试
   至少补一条受影响查询或更新路径的测试，确保结构变化没有破坏既有行为。

## Liquibase Changeset 格式约定

Schema 变更建议按 `VYYYYMMDD_NNN__description.sql` 命名放入 `migrations/`；数据脚本建议按相同命名规则放入 `data/`。每个 `changeSet` 都应提供对应回滚策略。

格式模板：

```sql
-- changeset [author]:[id]
-- rollback [rollback sql];
```

示例：

```sql
-- changeset kzou227@qq.com:202304061134
create table person(
  firstname    varchar(255)  not null,
  lastname     varchar(255)  not null
);
-- rollback drop table person;
```
