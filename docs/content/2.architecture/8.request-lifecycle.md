## 选用示例

本页用“功能开关分页查询/更新”这条链路说明 Mahu 的代码组织方式，因为它同时覆盖了：

- OpenAPI 契约
- 管理端 Controller
- 管理端 Service
- Repository
- Entity / Query
- 延时任务联动

## 1. OpenAPI 契约

功能开关接口首先定义在 `openapi/admin/openapi.yaml` 中，并通过拆分的 `paths/sys_feature-flags*.yaml` 文件描述具体接口。

这一步定义的是：

- 路由
- 请求参数
- 请求/响应模型
- 接口分组

生成命令由 `mahu-admin:mahu-admin-web:openApiGenerate` 驱动，产物写入 `mahu-admin/mahu-admin-web/src/main/gen`。

## 2. Controller 作为 HTTP 适配层

手写 Controller 位于：

- `mahu-admin/mahu-admin-web/src/main/java/cool/houge/mahu/admin/web/controller/sys/FeatureController.java`

这个类的主要职责只有三件事：

1. 从 `ServerRequest` 读取 query/path/body 参数。
2. 调用 `FeatureFlagService`。
3. 将领域对象映射为 OpenAPI 响应对象并返回。

不要把复杂业务规则堆在这一层，这也是 `mahu-admin-web` 保持可维护的关键。

## 3. Service 承担业务编排

功能开关服务位于：

- `mahu-admin/mahu-admin-infra/src/main/java/cool/houge/mahu/admin/sys/service/FeatureFlagService.java`

这一层负责：

- 根据 ID 获取实体并处理未找到场景。
- 创建和更新功能开关。
- 在 `enableAt` / `disableAt` 发生变化时，调用共享能力入队延时任务。

可以把它理解为“管理端语义下的业务编排层”。

## 4. Repository 承担数据库访问

对应数据访问层在：

- `mahu-dal/src/main/java/cool/houge/mahu/repository/sys/FeatureFlagRepository.java`

这里负责：

- 根据 `FeatureFlagQuery` 组装分页查询条件。
- 使用 EBean QBean 进行过滤与排序。
- 通过条件更新实现“到点启用 / 到点关闭”这类并发安全操作。

也就是说，Repository 处理的是“怎么安全高效地读写数据库”，而不是“接口层怎么组织响应”。

## 5. Entity 与 Query 定义领域模型

相关领域对象位于：

- Entity：`mahu-entity/src/main/java/cool/houge/mahu/entity/sys/FeatureFlag.java`
- Query：`mahu-model/src/main/java/cool/houge/mahu/query/sys/FeatureFlagQuery.java`

这里定义的是：

- 表结构映射
- 字段含义
- 查询条件承载对象

它们不关心 HTTP，也不直接表达页面行为。

## 6. 与任务系统的联动

功能开关更新后，如果存在未来生效时间，`FeatureFlagService` 会通过共享服务写入 delayed task。

后续由 `mahu-task` 中的 worker 拉起处理：

- Worker：`DelayedTaskDispatcherWorker`
- Handler：`FeatureFlagDelayedTaskHandler`

这体现了 Mahu 的一个重要设计：

- 同步请求只负责保存意图。
- 到点执行由后台任务系统完成。

## 整体时序

```text
OpenAPI 规范
  -> 生成 H*Service / VO
  -> FeatureController
  -> FeatureFlagService
  -> FeatureFlagRepository
  -> PostgreSQL
  -> 如有定时启停需求，入队 delayed task
  -> mahu-task worker 到点处理
```

## 改这个功能时通常要动哪些地方

### 新增字段

通常需要同时更新：

1. `mahu-db` 迁移脚本
2. `FeatureFlag` 实体
3. OpenAPI schema
4. bean mapper
5. Controller / Service / Repository 中的读写逻辑
6. 测试

### 新增筛选条件

通常需要同时更新：

1. OpenAPI query 参数
2. Controller 参数解析
3. `FeatureFlagQuery`
4. `FeatureFlagRepository.findPage(...)`
5. 对应测试

## 读代码建议

如果你第一次追某个功能，建议按下面顺序读：

1. `openapi/admin/openapi.yaml`
2. 对应 `paths/*.yaml`
3. 手写 Controller
4. 管理端 Service
5. Repository
6. Entity / Query
7. 如涉及异步，再看 `mahu-task`
