sys 模式包含系统相关的表，主要用于用户管理、认证、权限控制、日志记录、定时任务等功能。

## 表结构设计
### admins（员工表）

|--------|----------|------|------|
| id | serial | PRIMARY KEY | 主键，自增序列 |
| created_at | TIMESTAMP | | 创建时间 |
| updated_at | TIMESTAMP | | 更新时间 |
| deleted | BOOLEAN | DEFAULT FALSE | 软删除标志 |
| username | VARCHAR(128) | UNIQUE INDEX (where deleted = FALSE) | 用户名 |
| password | VARCHAR(1024) | | 登录密码 |

| avatar | VARCHAR(128) | | 头像地址 |
| status | SMALLINT | | 用户状态 |
| gender | SMALLINT | | 性别（0：其他，1：男性，2：女性） |

- `admin_username_uidx`：在 username 字段上创建唯一索引，条件为 deleted = FALSE

## sys 模式表关系

### 核心关系
1. **用户-角色关系**：
   - `admins` 表与 `roles` 表通过 `admin_roles` 关联表建立多对多关系
   - 一个管理员可以有多个角色，一个角色可以分配给多个管理员

2. **日志关联**：
   - `admin_access_logs`、`admin_auth_logs`、`admin_change_logs` 都通过 `admin_id` 关联到 `admins` 表

3. **定时任务**：
   - `scheduled_tasks` 表存储定时任务定义
   - `scheduled_task_logs` 表记录定时任务的执行历史

### 功能模块划分
1. **用户管理**：admins, roles, admin_roles
2. **认证管理**：auth_clients
3. **功能控制**：features
4. **任务调度**：scheduled_tasks, scheduled_task_logs, delayed_tasks
5. **日志记录**：admin_access_logs, admin_auth_logs, admin_change_logs
6. **通用存储**：objects, id_photos
